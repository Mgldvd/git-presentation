Problems with not having version control.

  • Trouble following the code.
  • Having folders named with: version 1, version 2, version 3
  • Difficulty in collaboration

Centralized vs Distributed

Centralized

  • Everyone must connect to a central server.
  • You can't work offline.
  • Writing policies.
  • Changes affect everyone.
  • Creating branches is very expensive, time-consuming or tedious.
  • Branches affect everyone because they are global.
  • You'd need naming policies so as not to interfere with others.
  • You wait until you have something stable to upload to the repository.
  • You never create something stable on the first try.

Distributed

  • They're all equally important.
  • You can work offline.
  • You must gain confidence and trust in others.
  • Changes only affect you.
  • Collaboration - Make changes without affecting others.
  • Experiment and branch out the code as you wish.
  • You don't have to ask for write permission.
  • There is no single point of failure, and the information is naturally replicated.

Git Qualities

  • Designed to handle large projects.
  • It's extremely fast.
  • Cryptographic authentication of history.
  • Very simple and compact file format.
  • 100% distributed.
  • Can be synchronized by any means.

Git is for the big ones

  • Linux kernel has more than 22,000 files.
  • Linux has been using Git for several years, averaging 4.5 merges per day.
  • Showing the differences of the entire kernel, 74,000 commits, takes about 7 seconds (cold cache) or 2.3 seconds (warm cache).
  • The CVS repository of the Mozilla Project is 3GB, in Subversion it is 12GB in fsfs format. In Git it's 300MB.

What is Git

Git is a distributed source code control system or SCM.

  • Knowing who's touched what and when.
  • Control over how our project has changed over time.
  • Going back to previous versions quickly.
  • Version control through labels.
  • Security: all internal data structures are signed with SHA1. You cannot change the code without detection.
  • It improves our ability to work as a team.
  • Extremely efficient merging and branching.
