Problemas de no tener control de versiones

  • Problemas para seguir el codigo.
  • Tener carpetas de version 1, version 2, version 3
  • Dificultad en la colaboración

Centralizado vs Distribuido

Centralizado

  • Todos deben conectarse a un servidor central.
  • No puedes trabajar sin conexión..
  • Políticas de escritura.
  • Los cambios afectan a todos.
  • Crear ramas es muy costoso, tardado o tedioso.
  • Las ramas afectan a todos porque son globales.
  • Necesitarías políticas de nomenclaturas para no estorbar a otros.
  • Te esperas a que tengas algo estable para subirlo al repositorio.
  • Nunca haces algo estable a la primera

Distribuido

  • Todos son igual de importantes.
  • Se puede trabajar sin conexión.
  • Debes ganar confianza y confiar en otros.
  • Los cambios solo te afectan a ti.
  • Colaboración - Realiza cambios sin afectar a los demás.
  • Experimenta y ramifica el código a tu gusto.
  • No tienes que pedir permisos de escritura.
  • No hay un punto débil central, y la información se replica de forma natural.

Cualidades de Git

  • Diseñado para manejar proyectos grandes.
  • Es extremadamente rápido.
  • Autentificación criptográfica del historial.
  • Formato de archivo muy sencillo y compacto.
  • 100% distribuido.
  • Se puede sincronizar por cualquier medio.

Git es para grandes

  • Linux kernel tiene más de 22,000 archivos.
  • Ha usado Git los últimos años, haciendo 4.5 merges al día.
  • Mostrar las diferencias de todo el kernel, 74,000 commits, toman aprox. 7 segundos (cold cache) 2.3 seg. (warm cache).
  • El repositorio en CVS del Mozilla Project es de 3GB, en Subversion es de 12GB en formato fsfs. En Git es de 300 Mb

Que es Git

Git es un sistema distribuido de control de código fuente o SCM (en inglés Source Code Management).

  • Saber quién ha tocado qué y cuándo
  • Control sobre cómo ha cambiado nuestro proyecto con el paso del tiempo
  • Volver hacia atrás de una forma rápida
  • Control de versiones a través de etiquetas.
  • Seguridad: todas las estructuras internas de datos están firmadas con SHA1. No se puede cambiar el código sin que nos enteremos
  • Mejora nuestra capacidad de trabajar en equipo
  • Merging y branching extremadamente eficientes
